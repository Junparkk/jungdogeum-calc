원본 HTML은 `var(--color-text-primary)`, `var(--border-radius-md)` 등 토스 인앱 디자인 토큰을 사용한다.

## 타입 정의

```ts
type Schedule = {
  id: number;
  name: string;
  date: string;      // YYYY-MM-DD
  amt: number;
};

type Payment = {
  id: number;
  schId: number;
  schName: string;
  date: string;      // YYYY-MM-DD
  amt: number;
  bulk: boolean;     // 월 단위 일괄 입력 여부
};

type AppState = {
  schedules: Schedule[];
  payments: Payment[];
  rate: number;      // 연 할인율 (기본 5)
};
```

## 핵심 계산 공식 (`lib/calc.ts`)

원본 HTML과 동일하게 이식:

- `calcCredit(amt, days, rate)` = `amt / (1 − rate × days/365)` — 수납인정금액
- `effectiveSimple(days, rate)` = `rate / (1 − rate × days/365)` — 단리 환산 실효이율
- `effectiveCompound(days, rate)` = `(1 / (1 − rate × days/365))^(365/days) − 1` — 연복리 환산 실효이율
- `rt ≥ 1`인 경우(할인율이 기간 대비 과도) 방어 처리는 원본과 동일

## 저장/복원 (`lib/storage.ts`)

- 키: `prepayment:state:v1`
- 값: `JSON.stringify({ schedules, payments, rate, schIdCounter, payIdCounter })`
- 런타임 분기:
  - 토스 인앱: `@apps-in-toss/web-framework`의 `Storage.setItem/getItem/removeItem` 사용 (비동기)
  - 로컬 dev / 일반 브라우저: `window.localStorage` 사용 (동기를 비동기로 래핑)
- `App.tsx`에서:
  - 마운트 시 `storage.load()`로 복원 (빈 상태면 기본값 사용)
  - `schedules`/`payments`/`rate` 변경마다 `storage.save()` 호출
  - 저장 실패는 콘솔 경고만 남기고 UI는 계속 동작

## 초기화 버튼

## 기능 체크리스트 (HTML에서 그대로 포팅)

- 중도금 일정 추가 (이름/기준일/금액 검증)
- 중도금 일정 삭제 — 삭제 시 연결된 납부 내역도 삭제
- 중도금 카드: 배지 + 기준일 + 금액 + 진행률 바 + 충당% + 잔여액
- 연 할인율 입력 (기본 5%, 실시간 재계산)
- 납부 내역 단건 추가 (중도금 선택 / 납부일 / 납부액 / 기준일 이전 검증)
- 납부 내역 월 단위 일괄 추가 (시작 월 / 종료 월 / 매월 납부일 / 월 납부액, 기준일 이후 자동 제외)
- 납부 내역 개별 삭제, 전체 삭제
- 4개 요약 카드: 실제 납부액 / 할인 후 충당액 / 총 할인액 / 전체 잔여액
- 실효이율 3카드 (가중평균 선납일수 기준): 표면 / 단리 / 복리
- 납부 내역 테이블: 중도금 / 날짜 / 납부액 / 선납일수 / 할인액 / 충당액 / 실효이율 / 삭제
- 날짜 정렬 (스케줄: 기준일, 납부 내역: 납부일)
- 경고 메시지 토스트 (입력창 하단 3초 후 자동 소거)
